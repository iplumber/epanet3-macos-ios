# EPANET macOS/iOS 功能规划 v1

## 1. 文档定位
- 本文档是产品级总控文档，用于统一功能范围、界面规范、交互规范与开发顺序。
- 本文档优先级高于零散讨论记录，后续开发、测试、验收均以本文档为准。
- 适用版本：v1。
- 当前状态：v1 主线阶段 A/B/C/D/E 已完成（含 E3 第二版）。

## 2. 约束与范围
- 平台范围：`macOS` + `iOS`。
- 文件格式范围：**仅支持 `.inp`**。
- 明确不做：`.net` 相关导入、导出、兼容转换。
- 本阶段目标：先固化 `macOS` 浅色/深色界面规范，再进入功能开发。

## 3. v1 核心功能清单
- 文件管理：新建、打开、关闭、保存、另存、导入/导出 `.inp`。
- 管网对象：节点、水池、水库、管段、阀门、水泵的增删改查。
- 视图控制：放大、缩小、平移、全屏、适配视图。
- 显示控制：图例设置、对象大小、颜色、文字标注开关。
- 参数设置：英制/公制单位、水力参数、计算参数（时长、步长、水损公式）。
- 计算与结果：运行计算、成功与报错信息、结果图层查看、对象结果查看。

## 4. macOS 界面设计（v1 第一优先）

### 4.1 设计基线
- 深色原型：`UI/epanet-macos.html`
- 浅色原型：`UI/epanet-macos-light.html`
- 功能细表：`UI/epanet_feature_detail_table.html`

### 4.2 信息架构
- 顶部标题栏：当前文件名、应用名。
- 顶部工具栏：编辑模式、撤销重做、图例/标注、参数、运行计算。
- 左侧边栏：图层树、对象统计、搜索与过滤、计算状态摘要。
- 中央画布：管网渲染层、结果叠加层、缩放控件、图例面板。
- 右侧检查器：属性/结果/样式三标签页。
- 底部状态栏：计算状态、对象总数、单位/公式、坐标与缩放信息。

### 4.3 浅色/深色一致性规则
- 两套主题仅替换颜色变量，不改变布局结构、控件位置与交互路径。
- 语义色一致：节点/水池/水库/阀门/水泵在浅色和深色主题下保持同一语义映射。
- 状态色一致：成功、警告、错误的色彩语义保持一致。
- 结果表达一致：压力/流量等结果色带区间定义保持一致，仅调整主题下对比度。

### 4.4 交互规范（macOS）
- 选择模式为默认模式，优先保证“选中-查看-编辑”闭环。
- 编辑对象时，检查器即时反映选中对象，并按对象类型显示可编辑字段。
- 运行计算后，结果自动叠加到画布，并在状态栏与 Toast 给出反馈。
- 报错时必须可定位对象（错误列表项点击后高亮并居中）。

## 5. iOS 交互原则（与 macOS 对齐，不展开视觉细节）
- 采用导航栏 + 底部操作栏 + Bottom Sheet 的触控优先结构。
- 保持与 macOS 的功能等价，不做功能阉割，仅调整交互形式。
- 属性与结果面板采用分段页签，编辑链路与字段定义与 macOS 对齐。

## 8. v1 实施结论（归档）
- 范围执行结果：保持“仅 `.inp`”约束，`.net` 不纳入 v1。
- 能力执行结果：已形成“打开 -> 编辑 -> 新增/删除 -> 参数设置 -> 运行计算 -> 结果上图 -> 错误定位”的最小闭环。
- 回归执行结果：`net1.inp`、`any1.inp`、`400.inp` 通过；`10000.inp`、`90000.inp` 完成性能与稳定性检查。
- 稳定性执行结果：E2 错误可用性自检与 E3 UI 稳定性回归清单已落地。
- 阶段结论：v1 达到可验收与可交接状态，可进入发布归档或 v1.x 增强阶段。
