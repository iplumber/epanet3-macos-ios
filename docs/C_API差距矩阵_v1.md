# C API 差距矩阵 v1（阶段 B1）

## 1. 目标
- 明确引擎层 C API 与桥接层在 v1 目标下的能力差距。
- 为阶段 B2/B3 提供可执行顺序。

## 2. 范围
- 仅支持 `.inp` 工作流。
- 聚焦对象 CRUD、参数读写、错误信息标准化。

## 4. 阶段 B 后续顺序

### B2 第二批（建议优先）
1. `EN_setNodeValue`
2. `EN_setLinkValue`
3. `EN_createNode` / `EN_createLink`
4. `EN_deleteNode` / `EN_deleteLink`
5. Swift Bridge 对应封装

### B3 错误标准化
1. `EN_getError` 最小可用实现（错误码 -> 文本）
2. Swift 侧统一错误结构（错误码、文本、对象定位）
3. UI 层错误展示统一入口

## 5. 当前阶段结论
- 阶段 B1：完成。
- 阶段 B2：已完成第一批（参数与时间读写）；下一步进入对象编辑 CRUD 能力补齐。
