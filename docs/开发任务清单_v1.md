## 3. 模块责任划分

### 3.1 引擎层模块
- 目标：提供可编辑、可计算、可定位错误的底层能力
- 关注：对象 CRUD、参数读写、错误码一致性

### 3.2 桥接层模块
- 目标：把引擎能力稳定暴露给 Swift 层
- 关注：接口清晰、错误可读、调用一致

### 3.3 界面与状态模块
- 目标：形成 macOS 可用编辑闭环
- 关注：状态机、交互一致性、结果反馈

### 3.4 测试与验收模块
- 目标：每阶段可验证、可回归
- 关注：算例回归、错误场景、文档同步
