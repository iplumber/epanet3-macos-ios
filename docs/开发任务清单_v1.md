### 6.5 视窗策略与后续自有格式（产品约定 + v2+ backlog，**下一版本再开发，当前不开展**）

**数据边界**

- `.inp` 仅承载水力/拓扑等 **EPANET 标准语义**；**不向 `.inp` 写入** 画布视窗（缩放、平移）、窗口几何、侧栏布局等 UI 状态。

- 打开或切换到某一 `.inp` 时，画布 **默认按 `NetworkScene` 的坐标 bounds（含现有 padding）全貌显示**（`userScale = 1`、`pan = 0`），**不恢复** 上次会话的 scale/pan。
- 实现参考：`EPANET3App/EPANET3App/ContentView.swift` 中 `loadedSceneRoot` 对 `filePath` 的 `.task(id:)` 重置逻辑（已移除 `CanvasViewStatePersistence`）。

- 规划一种 **自有容器格式**（扩展名、版本字段、序列化方式 **TBD**），用于承载 **非 `.inp` 标准** 的信息，例如：

## 7. `.inp` 保存一致性（保留式合并）

**问题**：使用 `ProjectWriter` 全量重写会在保存时丢掉原文件中的注释、未知章节与章节顺序。

**实现**：打开成功且存在完整 `project` 时，在内存中保留全文快照 `inpSourceSnapshot`。属性面板编辑基础字段后（短防抖后）自动把**用户实际改动的字段**记入 `inpPendingSaveDelta`（并只对引擎调用对应 `set*`）；无需单独点面板「保存」。菜单「保存 / 另存为」调用 `InpPreservingSaver.applyPatches`，**仅替换**快照里对应 ID 的对应 token（如只改管径则只改 `[PIPES]` 行上的管径列），其余字符与行不变；若无待写增量则**不写磁盘**。增删节点/管段、修改计算参数（Options/Times）后快照作废，下次保存走全量 `ProjectWriter`。

**手工验证（建议 `any1.inp`）**：
1. 备份：`cp any1.inp any1.inp.bak`
2. 在应用中打开 `any1.inp`，在属性面板只改一条管段的管径，稍等防抖结束（或失焦后），再使用菜单「保存」写盘。
3. 对比：`diff -u any1.inp.bak any1.inp` —— 应仅有该管段行上管径数值变化（长度/糙率列应与备份一致），章节与注释不重排。
4. 可选：`wc -l any1.inp.bak any1.inp` 行数应相同或仅有空白行差异。
